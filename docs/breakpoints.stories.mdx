# Брейкпоинты

Контрольные точки для медиа запросов задаются в [mq.json](https://github.com/core-ds/core-components/blob/master/packages/mq/src/mq.json).

<Markdown>
    {`
| Название       | Значение        |
| -------------- | --------------- |
| --mobile-s   | 320px to 100%   |
| --mobile-m   | 375px to 100%   |
| --mobile-l   | 412px to 100%   |
| --mobile     | 0 to 599px      |
| --tablet-s   | 600px to 100%   |
| --tablet-m   | 768px to 100%   |
| --tablet     | 600px to 1023px |
| --desktop-s  | 1024px to 100%  |
| --desktop-m  | 1280px to 100%  |
| --desktop-l  | 1440px to 100%  |
| --desktop-xl | 1920px to 100%  |
| --desktop    | 1024px to 100%  |`}
</Markdown>

## Пример использования в CSS

Требования: версия `arui-scripts` выше `10.2.0`

```css
@media (--mobile-m) {
    /* styles */
}

@media (--desktop-s) {
    /* styles */
}
```

## Пример использования в JS

При необходимости рендеринга по условию, используйте компонент [Mq](?path=/docs/mq--docs), который основан на `window.matchMedia API`:

```jsx
import { Mq, useMatchMedia } from '@alfalab/core-components/mq';

const Component = () => (
    <Mq query='--mobile'>
        <Button>Mobile button</Button>
    </Mq>
);

const Component = () => {
    const query = '--mobile';
    const [matches] = useMatchMedia(query);
    return `Matches ${query}: ${matches}`;
};
```

## Пример изменения breakpoint в core компонентах

По-умолчанию в `core-components` респонсивные компоненты переключаются на десктопный вид, если ширина вьюпорта превышает 1024px.
Для переопределения этого поведения все респонсивные компоненты имеют проп `breakpoint`.

```jsx
    <Select
        // Селект переключится в десктопный вид, если ширина вьюпорта >= 768
        breakpoint={768}
        ...
```

## Глобальные брейкпоинты
Как уже было сказано, по умолчанию брейкпоинты в компонентах установлены на `1024px`.
Это значение можно переопределить глобально. Для этого необходимо подключить в корневой CSS файл необходимый брейкпоинт.
Этот вариант подходит для команд, которые не вырезают `CSS` переменные из бандлов.

```css
/* index.css */

@import "@alfalab/core-components/themes/breakpoints/breakpoint-768.css";
```

Альтернативным вариантом подключения глобального брейкпоинта является импорт в корневой `ts/tsx` файл вашего приложения.
```jsx
/* app.tsx */

import '@alfalab/core-components/shared/customBreakpoints/injectCustomBreakpoint-768';
```

_*На данный момент доступно переопределение только на `768px`. Если у вас есть специфический случай и вам необходимо добавить свой собственный брейкпоинт, свяжитесь с командой разработки или отправьте pull request со своими правками._
